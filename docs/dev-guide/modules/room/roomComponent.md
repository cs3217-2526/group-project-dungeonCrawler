# Room Component

A room in the dungeon is represented as an ECS entity. The entity holds a `RoomComponent` that describes everything about the room's layout — its bounds in world space, doorways, and spawn points.

## RoomComponent

```swift
public struct RoomComponent: Component {
    public let roomID: UUID          // Unique identifier for this room
    public var bounds: RoomBounds    // Position and size in world space
    public var doorways: [Doorway]   // Connections to adjacent rooms
    public var spawnPoints: [SpawnPoint]  // Where entities appear
    public var gridLayout: GridLayout?    // Optional tile grid (for grid-based generation)
}
```

Each room entity also gets a `TransformComponent` positioned at `bounds.center`, which allows spatial queries against the room's position in world space.

## RoomBounds

`RoomBounds` defines the rectangular area of a room in world coordinates (SpriteKit space: origin bottom-left, Y axis pointing up).

```swift
public struct RoomBounds {
    public var origin: SIMD2<Float>  // Bottom-left corner
    public var size: SIMD2<Float>    // Width and height
}
```

Key computed properties:

| Property | Description |
|---|---|
| `center` | `origin + size / 2` |
| `max` | Top-right corner: `origin + size` |
| `contains(_ point:)` | Returns `true` if the point falls within the rectangle (inclusive) |
| `randomPosition(margin:)` | A random point inset by `margin` from all edges. Default margin is `50`. |

### Sizing Rooms

Room bounds are built in `GameScene` and passed to `MapSystem`. The bounds are calculated from the screen size so the room fills the visible area:

```swift
let roomWidth  = Float(size.width)  * 0.9
let roomHeight = Float(size.height) * 0.9
let bounds = RoomBounds(
    origin: SIMD2<Float>(-roomWidth / 2, -roomHeight / 2),
    size:   SIMD2<Float>(roomWidth, roomHeight)
)
```

The inset ensures the outer edge of each perimeter wall sits flush with the screen boundary rather than extending past it.

## Doorway

A `Doorway` describes a connection point on the room's perimeter.

```swift
public struct Doorway {
    public var position: SIMD2<Float>     // World position of the doorway centre
    public var direction: Direction        // Which wall it's on
    public var width: Float               // Opening width (default 64)
    public var connectedRoomID: UUID?     // The room on the other side (nil if unconnected)
    public var isLocked: Bool             // Whether the door blocks passage
}
```

`Direction` provides a unit vector for each cardinal direction, used to offset spawn points inward from the doorway:

```swift
public enum Direction {
    case north  // vector: (0,  1)
    case south  // vector: (0, -1)
    case east   // vector: (1,  0)
    case west   // vector: (-1, 0)
}
```

## SpawnPoint

A `SpawnPoint` marks a position within the room where an entity should appear.

```swift
public struct SpawnPoint {
    public var position: SIMD2<Float>
    public var type: SpawnType
    public var isUsed: Bool

    public enum SpawnType {
        case playerEntry  // Entry point for the player
        case enemy        // Spawn point for an enemy
    }
}
```

Spawn points are generated by `MapSystem` and stored on `RoomComponent.spawnPoints`. They are not ECS entities — they are plain data attached to the room.

## Room State Tags

Room state is tracked with tag components, which can be added or removed to transition a room between states:

| Tag | Meaning |
|---|---|
| `RoomLockedTag` | Exits are locked; player cannot leave |
| `RoomInCombatTag` | Enemies are active in the room |

Planned (currently commented out):

| Tag | Meaning |
|---|---|
| `RoomClearedTag` | All enemies defeated; exits unlocked |
| `RoomVisitedTag` | Player has entered this room before |
| `RoomActiveTag` | The room the player is currently in |

## Map Entity Tags

Individual geometry entities within a room are marked with tags so systems can query them by type:

| Tag | Entity type |
|---|---|
| `WallTag` | Perimeter wall segment |
| `FloorTag` | Floor fill entity |
| `ObstacleTag` | Interior obstacle |
| `DoorTag` | Doorway entity (holds `direction`) |
| `LockedDoorTag` | Applied to a locked door entity |

## GridLayout

`GridLayout` is an optional field on `RoomComponent` for tile-based room generation. It represents the room as a 2D grid of typed cells.

```swift
public struct GridLayout {
    public var gridSize: SIMD2<Int>      // Number of cells in X and Y
    public var cellSize: Float           // World units per cell
    public var tiles: [[TileType]]       // [row][col], initialised to .floor
}

public enum TileType: Int {
    case floor    = 0
    case wall     = 1
    case obstacle = 2
    case pit      = 3
}
```

`GridLayout.worldPosition(gridX:gridY:roomOrigin:)` converts a grid coordinate to a world position, placing the entity at the centre of the cell:

```swift
let pos = grid.worldPosition(gridX: 2, gridY: 3, roomOrigin: bounds.origin)
// x = origin.x + 2 * cellSize + cellSize / 2
// y = origin.y + 3 * cellSize + cellSize / 2
```

Grid-based generation is implemented in `RoomGenerator.generateGridBasedRoom(room:world:)` but is not currently used by the default generation path. To opt in, pass `useGrid: true` to `EntityFactory.makeRoom`.
